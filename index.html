Modules · Level 2

各模組細節

每個模組展開後的關鍵設計決策與實作細節。

Module A · TQC 巡航

patrol_node / burger_sac_env

  • TQC（Truncated Quantile Critics，sb3-contrib）抑制 Q overestimation — SAC 後繼者
  • 觀測空間 744 維 = (180 LiDAR beams + 6 state) × frame stack K=4
  • 1D-Conv LiDAR encoder + LayerNorm，能學「局部 spatial pattern」（門框、走廊）
  • Reward = γΦ(s') − Φ(s)（Ng-Harada-Russell 1999 potential-based shaping）+ smooth + sparse
  • Curriculum 1→5 waypoints 自動升級；Domain Randomization + 5% 對抗訓練
  • 動態巡邏點：/patrol/goto 帶 HMAC 簽章 + 座標 ±2.5m 範圍檢查
  • 收到 /security/alerts 立刻 v=0, ω=0；resume timer 不 reset on alert，加 cascade quiet window
Module B · 資安監控

monitor_node + 簽章鏈

  • HMAC envelope v3：{body{channel, nonce, payload, ts}, sig} — 防偽造 + 防重放 + 防跨頻道
  • 接收端 ReplayCache（LRU + TTL eviction），訊息 max_age 3 秒拒絕過期
  • 檔案簽章 .sha256.hmac：模型 / buffer load 前驗章，篡改 sys.exit(2) 拒絕
  • 節點偵測：開機 baseline 快照 + 15s grace period 只吸收白名單，移除 prefix 後門
  • 敏感參數 ParameterDescriptor(read_only=True)：whitelist / poll_interval 不可被改
  • LINE 推播 30s batch 聚合，防 alert 洪水洗版
  • SROS2 Permissive 模式運行中；Enforce migration 列入 90 天計畫
Module C · 任務管理

Mission / Status / Sensor Hub

  • mission_manager：訂閱警報，管理任務狀態機（巡邏 / 停止 / 恢復）；綁 channel=CH_MISSION
  • system_status：對外廣播健康度；self-watch 改為 log-only + rate-limit cooldown（避免 N13 confused-deputy）
  • sensor_hub：LiDAR + IMU 融合分發；綁 channel=CH_SENSOR 簽章發送
  • 全部下游模組以 verify_alert(expected_channel=...) 驗章，cross-channel 攻擊被擋
Module D · 紅隊 + 行為 IDS

intelligent_defense_node + 紅隊 PoC

  • D1 物理門檻（cmd_vel > 0.23 m/s 視為攻擊；Burger 上限 0.22）
  • D2 方向衝突 / D3 scan 重複 / D4 unauthorized publisher（白名單 + 計數）
  • D5 heartbeat 看門狗（10s 超時告警）
  • D6 cmd-vs-odom-vs-scan 行為一致性偵測（odom 移動但 scan 靜止 = 矛盾 = 攻擊）
  • 紅隊 PoC N1–N24 獨立可執行檔（heartbeat replay / channel confusion / pickle RCE / cascade DoS）
  • pytest tests/test_security.py — 24 個自動化測試全綠
  • 偵測即攻擊面教訓：cascade DoS 偵測 → 90s 內 2 次 pause → quiet window 升級到外部介入
